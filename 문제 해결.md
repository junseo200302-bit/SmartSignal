# Problems and Solutions

## 1. Missing Python Packages

When I first tried to run the application, Streamlit could not start because some required Python packages were not installed.

To solve this problem, I installed the missing libraries and created a requirements.txt file so that the deployment environment could automatically install all dependencies.

---

## 2. Simulation Speed Adjustment

During testing, the vehicle movement looked too slow and made it difficult to demonstrate the traffic flow clearly.

I adjusted the animation timing and increased the simulation speed so that traffic movement could be observed more easily during presentations.

---

## 3. Signal Timing Logic

In an early version of the project, green-light time was distributed too evenly across all directions. As a result, congested roads did not receive enough signal time.

To improve this, I implemented a weighted demand calculation that considers vehicle volume, pedestrian demand, and emergency vehicle priority when allocating green-light duration.
